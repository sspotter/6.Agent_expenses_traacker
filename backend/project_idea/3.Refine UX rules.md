Good UX rules will make this feel **fast, safe, and reliable**, especially since this is a **daily-use financial tool**.

# 1. Core UX Principles (Guiding Rules)

These apply everywhere:

### 1️⃣ Minimize daily effort

- Marking today’s collection should take **1 click**
    
- Amount input only when needed (partials)
    

### 2️⃣ Prevent financial mistakes

- Never overwrite data silently
    
- Always show before/after amounts
    

### 3️⃣ Make partials obvious

- Partial ≠ completed
    
- Partial must always show amount
    

### 4️⃣ History is sacred

- Past months are read-only
    
- No auto-modification across days
    

---

# 2. Daily Cell UX Rules

### Default state

- Empty cell = no action taken yet
    
- Today’s cell is **highlighted subtly**
    

---

### One-click full collection

**Action**

- Click today’s cell → instant ✅ Full
    

**Rule**

- System assumes **expected amount collected**
    
- No modal shown
    
- Undo available
    

---

### Partial collection flow

**Action**

- Long click / double click / “Edit” icon
    

**Flow**

1. Modal opens
    
2. User enters collected amount
    
3. Status auto-selected as **Partial**
    

**Rules**

- `0 < collected < expected` → Partial
    
- Amount is mandatory
    
- Display `collected / expected` in cell
    

---

### Missed day

**Action**

- Mark explicitly as Missed (❌)
    

**Rules**

- Missed requires confirmation
    
- Reason optional but encouraged
    
- Cannot be auto-marked
    

---

### Undo & edit

**Rules**

- Same-day edits allowed freely
    
- Previous days (current month) show warning:
    
    > “Editing past days affects reports”
    

---

# 3. Visual UX Rules (Very Important)

### Cell appearance priority

1. Status color
    
2. Amount (if partial)
    
3. Icon
    

Example:

```
🟡
50
```

Not:

```
🟡 50/100 ✔
```

Avoid clutter.

---

### Color consistency

- Green = Full
    
- Yellow = Partial
    
- Red = Missed
    
- Gray = No data
    

Never change meanings.

---

# 4. Weekly Expenses UX Rules

### Adding expenses

**Action**

- Click “+” under week
    

**Rules**

- Expense must have:
    
    - Amount
        
    - Category (optional)
        
- Date range defaults to week
    

---

### Expense visibility

- Weekly totals update instantly
    
- Monthly summary updates instantly
    

No recalculation delays.

# 5. Monthly Summary UX Rules

### Real-time feedback

- Summary updates **live**
    
- Outstanding amount animates subtly on change
    

---

### Drill-down

Clicking:

- “Partial (7)” → highlights all partial days
    
- “Outstanding” → tooltip explaining breakdown

# 6. Handling Partial Payments Across Days (Key UX)

This is where confusion often happens — here’s the clean rule:

### UX Rule

❌ **No automatic carry-over UI**

Why?

- Carry-over creates confusion and hidden state
    
- Users lose trust
    

### Instead:

- Each day is independent
    
- Outstanding amount explains the gap
    

### Optional enhancement (advanced)

Show a **“Cumulative Deficit”** line in monthly summary:

```
Outstanding: 450
(From 7 partial + 6 missed days)
```

---

# 7. Previous Months UX Rules

### Read-only mode

- All inputs disabled
    
- Hover shows tooltip:
    
    > “Past months are locked”
    

### Exception

- Admin / override mode (optional later)
    

---

# 8. Error Prevention & Feedback

### Input validation

- Collected amount cannot exceed expected
    
- Negative values blocked
    

### Confirmation rules

- Deleting an entry → confirmation
    
- Editing a partial to full → show difference
    

---

# 9. Speed Optimizations (Daily User Friendly)

### Keyboard shortcuts (optional)

- `Enter` → mark full
    
- `P` → open partial modal
    
- `M` → mark missed
    

---

### Auto-focus

- Amount input auto-focused
    
- Save on Enter
    

---

# 10. UX Edge Cases (Often Forgotten)

### Multiple partial days in a row

✔ Allowed  
✔ Each day shows its own partial  
✔ Monthly outstanding shows accumulation

---

### Full day after partial days

✔ No special treatment  
✔ Green full day stays green  
✔ No auto-correction

---

### Month with fewer than 31 days

- Invalid days hidden
    
- Not disabled (no dead cells)
    

---
