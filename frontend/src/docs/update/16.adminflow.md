# Admin Page – Implementation Steps

## Phase 0: Foundations (Do First)

### Step 0.1 — Routing & Access

**Goal:** Admin page is reachable and protected.

* Route: `/admin`
* Guard:

  * `role === 'admin'` OR `is_admin === true`
* Redirect non-admins → worker page or login

**UI**

* Empty Admin shell
* Top banner: **“Admin Dashboard”**

---

### Step 0.2 — Admin Layout Skeleton

**Goal:** Stable layout that never re-renders away.

**Components**

```
AdminPage
 ├─ AdminHeader (global metrics + actions)
 └─ WorkersSection
```

**UI**

* Sticky header
* Scrollable content area

---

## Phase 1: Global Admin Header

### Step 1.1 — Worker Metrics

**Display**

* Total workers
* Active workers count
* Suspended workers count

**Data**

* `workers[]`
* Derived counts

**UI**

```
Total Workers: 24
Active: 20 | Suspended: 4
```

---

### Step 1.2 — Global Financial Totals

**Display**

* Expected
* Collected
* Outstanding

**Data**

* Aggregated from all workers
* Respect suspended toggle (future)

**UI**

```
Expected: $12,000
Collected: $9,500
Outstanding: $2,500
```

---

### Step 1.3 — Global Actions

**Buttons**

* ➕ Add Worker
* ⬇ Export Data
* ⬆ Import Data
* ⚙ Settings (disabled placeholder)

**Rules**

* Export / Import require confirmation
* No silent actions

---

## Phase 2: Workers List

### Step 2.1 — Sectioning

**Display**

* **Active Workers** (expanded)
* **Suspended Workers** (collapsed)

**Behavior**

* Toggle expand/collapse
* Suspended section visually muted

---

### Step 2.2 — Worker Collapsed Card

**Display per worker**

* Name
* Status badge
* Current rate
* Monthly snapshot:

  * Expected
  * Collected
  * Outstanding
  * Completion %

**UI Intent**

* Fast scanning
* No edits here

---

## Phase 3: Expanded Worker View (Admin Mode)

### Step 3.1 — Expand / Collapse Behavior

* Click worker card → expands inline
* Only one expanded at a time (recommended)

---

### Step 3.2 — Worker Control Panel

**Display**

* Name
* Status (Active / Suspended)
* Password state (Set / Removed)
* Current rate

**Actions**

* Suspend / Reactivate
* Remove password
* Reset password
* “View as Worker” (admin mode banner)

⚠️ All destructive actions → confirm modal

---

## Phase 4: Rate Management (Critical Path)

### Step 4.1 — Show Active Rate

**Display**

```
Current Rate: $100 / day
Effective since: Jan 10
```

---

### Step 4.2 — Change Rate Modal

**Inputs**

* New rate
* Effective date (default = today)
* Optional note

**Rules**

* No editing old rates
* No recalculation of past data

**Data**

* Insert new rate history row

---

## Phase 5: Financial Overview (Per Worker)

### Step 5.1 — Monthly Summary

**Display**

* Expected
* Collected
* Outstanding
* Net balance

**Data**

* Computed using rate history
* Month selector (future-safe)

---

### Step 5.2 — Admin Adjustments

**Actions**

* Add manual adjustment (+ / –)
* Correct payment
* Correct expense

**Rules**

* Explicit reason required
* Logged
* Never silent overwrite

---

## Phase 6: Payments & Expenses Snapshot

### Step 6.1 — Recent Items

**Display**

* Last N payments
* Last N expenses

Each row:

* Amount
* Date
* Type
* Edit / Delete (admin only)

---

### Step 6.2 — Outstanding Highlighting

* Unpaid / unsettled items visually flagged

---

## Phase 7: Suspended Workers Rules

### Step 7.1 — Visibility

* Appear only in:

  * Admin page
  * Reports → Suspended section

### Step 7.2 — Interaction

* Data read-only
* Reactivation enabled
* No new payments unless reactivated

---

## Phase 8: Admin Mode UX

### Step 8.1 — Admin Banner

When viewing worker as admin:

```
Viewing Worker: John (Admin Mode)
[← Back to Admin]
```

* Always visible
* Clear exit path

---

## Phase 9: Export / Import

### Step 9.1 — Export

**Export includes**

* Workers
* Rate history
* Payments
* Expenses
* Settlements

**Metadata**

* Schema version
* Timestamp

---

### Step 9.2 — Import

**Safeguards**

* Schema check
* Dry run
* Explicit overwrite confirmation

---

## Phase 10: Security & Integrity

### Step 10.1 — Audit Logging

Log:

* Rate changes
* Financial edits
* Suspensions
* Imports / exports

---

## Final Definition of Done (Checklist)

✅ Admin route works
✅ Workers split active / suspended
✅ Rate changes don’t affect history
✅ Financial edits are explicit & logged
✅ Export / import reliable
✅ Admin UX clearly marked
