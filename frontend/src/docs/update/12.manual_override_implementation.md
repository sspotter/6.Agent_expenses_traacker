# Manual Override & Flexible Rate Implementation

## Implemented Features

### 1. Daily Expected Amount Override
- **Database**: Added `is_expected_override` boolean column to `daily_collections`.
- **UI**: In the Daily Collection Modal, clicking the expected amount turns it into an editable input.
- **Logic**: Saving an edited amount updates the `expected_amount` field in the database and sets the override flag. This persists the specific expectation for that day regardless of global rate changes.
- **Indicator**: A "MANUAL" badge appears next to the expected amount label if it has been overridden.

### 2. Flexible Rate Changes (Admin)
- **UI**: In the Worker Modal (Edit Profile), changing the "Daily Rate Target" now exposes an **"Effective From Date"** picker.
- **Logic**: 
    - Defaults to "Today".
    - Admin can select a past or future date.
    - The system closes the previous rate record on the day before the new effective date, and starts the new rate on the effective date.
- **History Preservation**: Past days (prior to the effective date) continue to use the old rate history.
- **Independence**: Existing daily collection records are **never** automatically touched by this change. They retain their snapshotted or overridden amounts.

### 3. Accurate Reporting
- **Monthly Summary**: Now iterates through every day of the month.
    - If a daily record exists: Uses its stored `expected_amount` (whether original or overridden).
    - If no record exists: Queries the historical rate table (`getRateForDate`) to find the correct rate for that specific date.
    - Sums these up for a perfect "Total Expected" that respects history and overrides.

## Technical Details

- **Hook Updates**:
    - `useWorkerRates.updateRate(rate, effectiveDate)`: Now accepts optional date.
    - `useCollections.saveCollection`: Now accepts `expected_amount` and `is_expected_override`.
- **Component Updates**:
    - `CollectionModal`: Added local state for `currentExpected`, `isOverride`, and edit mode toggling.
    - `WorkerModal`: Added conditional rendering for date picker when rate is modified.

## How to Test

1.  **Override a Day**:
    - Click on a day (today or past/future).
    - Click the large Expected Amount number (e.g. $100).
    - Change it to $150.
    - Save.
    - Verify the grid card or summary reflects the change. Re-open to see "MANUAL" badge.

2.  **Change Rate Mid-Month**:
    - Go to Worker Management (Admin).
    - Edit current worker.
    - Change Daily Rate to $200.
    - Set Effective Date to 3 days from now.
    - Save.
    - Check the Grid: Days before that date (if empty) should show old rate (e.g. ghost text or just summary calc). Days after should show new rate. 
    - Note: The grid UI might need to update its "empty state" display logic to show changing expected amounts if we visualize them on empty cards (currently it might just show total summary changes).

3.  **Verify History**:
    - Check that changing the rate didn't change the status of days that were already settled or collected in the past.
