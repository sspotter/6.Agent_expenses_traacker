# 📊 Monthly Expected Breakdown – Professional Specification

## 🎯 Objective

Split the current **“Expected”** metric into **two clearly defined concepts**:

1. **Monthly Expected (Contractual)**
2. **Worked Expected (Actual Expected Based on Attendance)**

This avoids misleading totals and aligns finance, operations, and reporting.

---

## 🧩 Definitions (Very Important)

### 1️⃣ Monthly Expected (Contractual)

> The **maximum possible expected amount** for the month if the worker worked **every scheduled day**.

**Formula:**

```ts
monthly_expected = daily_rate × number_of_days_in_month
```

**Example:**

* Daily rate = `$100`
* Month = `31 days`

➡️ **Monthly Expected = $3,100**

✅ This value:

* Never changes
* Is independent of attendance
* Represents the **target / ceiling**

---

### 2️⃣ Worked Expected (Attendance-Based)

> The **expected amount based only on days the worker actually worked**, including partial days.

This reflects **what the worker is actually accountable for**.

---

## 📅 Day Classification Logic

Each day in the month must be classified as **one of three states**:

| Day Type        | Credit                     | Description      |
| --------------- | -------------------------- | ---------------- |
| **Full Day**    | 1.0 × daily_rate           | Worked fully     |
| **Partial Day** | partial_ratio × daily_rate | Worked partially |
| **Missed Day**  | 0                          | Did not work     |

### Partial Day Rule

* Partial days may be:

  * 0.25
  * 0.5
  * 0.75
    (or whatever your business rules define)

---

## 🧮 Worked Expected Calculation

```ts
worked_expected =
  (full_days × daily_rate) +
  (partial_days × daily_rate × partial_ratio)
```

### Example (Your Screenshot Scenario)

* Daily rate: `$100`
* Full days: `1`
* Partial days: `2` (assume 0.5 each)
* Missed days: rest of month

```ts
worked_expected =
  (1 × 100) + (2 × 100 × 0.5)
  = 100 + 100
  = $300
```

✅ **Worked Expected = $300**
✅ **Monthly Expected = $3,100**

## 🧑‍💻 Data Model (Suggested)

```ts
interface AttendanceSummary {
  full_days: number;
  partial_days: number;
  missed_days: number;
  partial_ratio: number; // e.g. 0.5
}

interface ExpectedSummary {
  daily_rate: number;
  days_in_month: number;
  monthly_expected: number;
  worked_expected: number;
  attendance: AttendanceSummary;
}
```
