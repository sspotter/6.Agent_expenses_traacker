# 📤 EXPORT & REPORTING SYSTEM

## Calendar, PDF, and CSV Exports

---

## 1. Purpose

Enable users to **export financial data** in human-readable and shareable formats:

* 📸 Calendar screenshots (visual proof)
* 📄 PDF reports (summary + details)
* 📊 CSV exports (analysis, Excel, accounting)

This system is **read-only** and must not modify data.

---

## 2. Export Types Overview

| Format     | Use Case                    |
| ---------- | --------------------------- |
| Screenshot | WhatsApp, quick sharing     |
| PDF        | Printing, reporting, audits |
| CSV        | Excel, accounting, backups  |

---

## 3. Export Scope & Selection

### 3.1 Month Selector

User must be able to:

* Select **any month (past or current)**
* Preview before exporting

```
[ ◀ ]  February 2026  [ ▶ ]
```

---

### 3.2 What Gets Exported

For a selected month:

#### Calendar (Visual)

* Daily income grid
* Status colors (FULL / PARTIAL / MISSED)
* Settlement indicators
* Expense markers

#### Data (Tabular)

* Daily income table
* Settlement table
* Expense table
* Expense payments table
* Monthly summary

---

## 4. Screenshot Export (Calendar)

### 4.1 What Is Captured

* Calendar grid only (no navigation, no buttons)
* Clean print layout
* Consistent aspect ratio

### 4.2 UX Flow

1. User selects month
2. Clicks **Export → Screenshot**
3. System:

   * Switches UI to *print mode*
   * Hides controls
   * Captures calendar container
4. User downloads PNG

### 4.3 Technical Notes (Web)

* Use DOM-based capture (e.g. html-to-canvas approach)
* Fixed width layout
* Light background forced

---

## 5. PDF Export (Primary Report)

### 5.1 PDF Structure (Order Matters)

```
Page 1
 ├── Header (Month, User, Date Generated)
 ├── Monthly Summary
 └── Calendar Screenshot

Page 2
 ├── Daily Income Table
 └── Settlement Table

Page 3
 ├── Expenses Table
 └── Expense Payments Table
```

---

### 5.2 Monthly Summary Section

Include:

* Expected income
* Collected income
* Settled later
* Income debt
* Expense total
* Expense debt
* Net position

---

### 5.3 Tables in PDF

All tables must include:

* Dates
* Amounts
* Status
* Notes (if any)

Tables must be:

* Paginated
* Readable when printed (A4)
* Left-aligned numbers

---

## 6. CSV Export

### 6.1 Export Options

User can export:

* All data for the month (ZIP or multiple CSVs)
* Or individual datasets

### 6.2 CSV Files Generated

| File                       | Contents             |
| -------------------------- | -------------------- |
| daily_collections.csv      | Daily income         |
| settlements.csv            | Settlements          |
| settlement_allocations.csv | Allocation breakdown |
| expenses.csv               | Expenses             |
| expense_payments.csv       | Payments             |
| monthly_summary.csv        | Derived values       |

---

### 6.3 CSV Rules

* ISO dates
* No formatting
* No derived totals except summary
* UTF-8 encoded

## 8. Print / Export Mode (Important)

When exporting:

* Disable animations
* Disable hover effects
* Force:

  * White background
  * Dark text
  * High contrast

Use a global `exportMode = true` flag.

---

## 9. Data Integrity Rules

* Exported data must be **read-only**
* Must reflect **derived values**, not stored ones
* Must match UI exactly

---

## 10. Security & Permissions

* User can export **only their data**
* No cross-user exports
* All exports happen client-side or via secured backend

---

## 11. Edge Cases

* Month with no data → empty calendar + summary
* Large tables → multi-page PDF
* Partial settlements across months → clearly labeled
