# ✅ Collection Model Consistency – Task Breakdown

## 🎯 Goal

Ensure that **daily expected amounts** remain accurate and traceable when:

* the **daily expected value is edited**
* **no payment is recorded**
* **partial / missed days exist**

The system must **never depend on payments to define expectation**.

---

## 🧱 Core Principle (Non-Negotiable)

> **Expected ≠ Collected**

* Expected amount is **attendance-based**
* Collected amount is **payment-based**
* They must be **stored independently**

---

## 🧩 Task Group 1 – Data Model Update

### 🔹 Task 1.1: Extend `daily_collections` table

Ensure the model supports expectation without payment.

#### Required fields

```ts
daily_collections {
  id
  worker_id
  date

  expected_amount   // NEW (required)
  collected_amount  // nullable
  status            // 'full' | 'partial' | 'missed'
  partial_ratio     // nullable (e.g. 0.5)

  created_at
  updated_at
}
```

📌 Rules:

* `expected_amount` is **always set**
* `collected_amount` can be `NULL`
* `missed` days must still have a row

---

## 🧩 Task Group 2 – Expected Amount Update Logic

### 🔹 Task 2.1: Update expected amount without payment

When user edits expected amount for a day:

#### Behavior

* Update `expected_amount`
* Do **NOT** create or modify payment
* Do **NOT** touch `collected_amount`

#### Pseudocode

```ts
update daily_collections
set expected_amount = new_value
where worker_id = X
  and date = Y
```

---

### 🔹 Task 2.2: Status auto-adjustment

Status must reflect expectation, not payment.

| Condition                    | Status    |
| ---------------------------- | --------- |
| expected_amount = 0          | `missed`  |
| expected_amount < daily_rate | `partial` |
| expected_amount = daily_rate | `full`    |

```ts
status =
  expected_amount === 0 ? 'missed'
  : expected_amount < daily_rate ? 'partial'
  : 'full'
```

---

## 🧩 Task Group 3 – Payment Independence

### 🔹 Task 3.1: Allow expected-only days

System must support:

✔ Expected amount set
✔ No payment recorded
✔ Day still counted in summaries

Example:

```
Expected: $100
Collected: NULL
Outstanding: $100
```

---

### 🔹 Task 3.2: Payment does NOT overwrite expected

When a payment is added:

* It **only affects** `collected_amount`
* It **never recalculates** `expected_amount`

🚫 Forbidden:

```ts
expected_amount = collected_amount
```
