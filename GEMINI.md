
# Project Overview

This project is an **Agent Expenses Tracker**, designed as a daily payment tracking system for small recurring collections. It supports partial payments, late settlements, and weekly expenses, with a strong audit trail. The development methodology emphasizes using Markdown (MD) files as the source of truth for project thinking, specifications, and design, with AI playing a key role in code generation, validation, and documentation updates.

## Key Functionalities

*   **Daily Collection Tracking:** Monitor expected versus collected amounts daily, accommodating full, partial, and missed payments.
*   **Settlements:** Allow for the settlement of past partial payments in future periods without altering historical data.
*   **Reporting:** Generate monthly summaries and weekly income/expense breakdowns.
*   **User Interface:** A structured interface including month navigation, monthly summary panels, a daily collection grid, and a weekly expense/totals row. Mobile view collapses the grid into week tabs.

## Development Methodology

The core development philosophy is to treat `.md` files in the `docs/` directory as living documents that drive the project. AI is utilized as a collaborative tool for:
*   Generating code from specifications.
*   Validating logic and identifying inconsistencies.
*   Safely updating documentation based on code changes.

The recommended workflow is: **Docs (MD) → AI → Code → Review → Update Docs → Repeat**.

# Development Conventions

*   **Source of Truth:** All project specifications, design decisions, and requirements must be meticulously documented in the respective Markdown files within the `docs/` directory. Code generated must align with these documents.
*   **Versioning:** Treat documentation files (`.md`) as version-controlled assets, similar to code. Commit changes with clear messages describing the update (e.g., "Define settlement model," "Refine UX rules for partial payments").
*   **AI Collaboration:** Leverage AI tools by providing context from the Markdown files in prompts for code generation, logic validation, and documentation updates. Always provide specific context and task scope to the AI.
*   **Atomic Changes:** When using AI for generation or updates, aim for small, focused tasks per prompt.
*   **Immutability:** Adhere strictly to the rule of not editing historical data. Past months' data should be read-only.
