
## “Bulk Weekly Expense Generator”

**Goal:**
Allow users to add the *same weekly expense* across all weeks of a month **in one action**, without manual repetition.

---

# 1. Button Placement & Labeling (Very Important)

### Placement

* **Above the Expenses section** (global context)
* Visible only when a month is selected

### Button Label (clear & safe)

✅ **“Add Weekly Expense for Month”**
❌ Avoid “Auto” or “Quick” (feels dangerous)

---

# 2. Click Flow (UX Safe & Predictable)

### On Click → Modal Opens

```
┌──────────────────────────────────────────┐
│ Add Weekly Expense for Month             │
├──────────────────────────────────────────┤
│ Month:        February 2026 (read-only)  │
│                                          │
│ Amount per week:   [  50.00  ]            │
│                                          │
│ Category (optional):                     │
│ [ Rent / Transport / Food / Custom ]     │
│                                          │
│ Apply to:                                │
│ ☑ Week 1   ☑ Week 2   ☑ Week 3           │
│ ☑ Week 4   ☑ Week 5 (if exists)           │
│                                          │
│ Behavior:                                │
│ ○ Create new expenses only               │
│ ○ Skip weeks with existing expense       │
│                                          │
│ Payment status:                          │
│ ● Unpaid (default)                       │
│ ○ Mark as paid today                     │
│                                          │
│ [ Cancel ]   [ Add Weekly Expenses ]     │
└──────────────────────────────────────────┘
```

---

# 3. Critical Behavior Rules (Non-Negotiable)

### Expense Creation Rules

* One **expense record per week**
* Amount is **identical** for each selected week
* Each expense:

  * Has its own ID
  * Owns its own payment lifecycle
  * Appears in Debt Explorer if unpaid

✔ No aggregation
✔ No shortcuts
✔ Fully auditable

---

# 4. What Happens If Expenses Already Exist?

This is where most systems break — so we define it clearly.

### Strategy Options (User-Controlled)

1️⃣ **Create new only**

* Skip weeks that already have an expense
* Safest default

2️⃣ **Replace existing** *(advanced, optional later)*

* Requires confirmation
* Deletes + recreates expense

💡 Recommendation:
**Start with option #1 only** to avoid destructive behavior.

---

# 5. Payment Handling (Important Distinction)

### Default

* All created expenses are **UNPAID**
* Appear immediately in:

  * Weekly expense totals
  * Monthly expense summary
  * Debt Explorer

### If “Mark as paid today” selected

* Create:

  * Expense record
  * Expense payment record (today)
* Payment is **not a settlement**
* Expense disappears from debt

# 7. Why This UX Is Correct

✔ Saves time
✔ No hidden logic
✔ No retroactive edits
✔ Matches accounting mental model
✔ Scales to multi-user later

---

# 8. Data Model Impact (Minimal but Clean)

No schema changes required **if you already have**:

* `weekly_expenses`
* `expense_payments`

Only required logic:

* Loop weeks → insert rows
* Optional payment insert

---

# 9. Edge Cases (Handled Explicitly)

| Case              | Behavior                         |
| ----------------- | -------------------------------- |
| Month has 5 weeks | Show Week 5                      |
| Month has 4 weeks | Week 5 hidden                    |
| Partial month     | Only valid weeks selectable      |
| Existing expense  | Skipped                          |
| Undo              | Manual delete (no auto rollback) |

---

# 10. Final UX Rule (Carry Forward)

> Bulk actions must **create many small correct records**,
> never one big smart one.
